DSA JOURNEY EXECUTION PLAN
(Beginner → Strong DSA Foundations)

--------------------------------------------------
CORE RULES (READ FIRST)
--------------------------------------------------

Rule 1: No Jumping
- Follow Phase 1 → Phase 6 strictly
- Do NOT jump to LeetCode early
- Do NOT skip questions

Rule 2: One Question at a Time
- Solve one question fully
- Then move to the next

Rule 3: Confusion Is Normal
- Confusion = learning
- Debugging = growth

--------------------------------------------------
DAY 0 — PREPARATION (10–15 Minutes)
--------------------------------------------------

1. Open VS Code
2. Open your DSA-Journey folder
3. Verify:
   - .md files → questions & documentation
   - .java files → implementation

Mindset:
"I will solve problems, not watch solutions."

--------------------------------------------------
HOW TO SOLVE ONE QUESTION (MANDATORY PROCESS)
--------------------------------------------------

Step 1: Read the Question
- Read from the .md file
- Understand input and output
- Do NOT code immediately

Step 2: Dry Run (VERY IMPORTANT)
- Use pen and paper
- Example:
  Input: 7
  7 % 2 = 1 → Odd

Step 3: Write Logic in Plain English
- Write comments inside the .java file
- Example:
  // Take input
  // Apply condition / loop / recursion
  // Print output

Step 4: Write the Code
- Convert logic to Java
- Keep code simple
- Avoid shortcuts

Step 5: Test Multiple Inputs
- Change input values:
  int n = 0;
  int n = 5;
  int n = -3;

Step 6: Mark Progress
- Create a progress file (example):
  Phase1_Progress.txt

- Track like this:
  [x] Q01
  [x] Q02
  [ ] Q03

--------------------------------------------------
PHASE-WISE EXECUTION PLAN
--------------------------------------------------

PHASE 1 — CONDITIONAL THINKING
Goal: Train logical decision-making

- Total questions: 50
- Daily target: 5–7 questions
- Time: 7–10 days

Outcome:
- Comfortable with if/else
- Can explain logic verbally

--------------------------------------------------

PHASE 2 — LOOPS & PATTERNS
Goal: Master iteration and flow control

- Focus first on:
  - Basic loops
  - Number-based logic
- Do pattern printing at the end

- Daily target: 5 questions
- Time: 10–12 days

Outcome:
- Loops become automatic
- Clear understanding of for/while

--------------------------------------------------

PHASE 3 — RECURSION
Goal: Learn thinking in self-reference

- Focus on:
  - Base case
  - Recursive call
- Dry run recursion on paper

- Daily target: 3–4 questions
- Time: 8–10 days

Outcome:
- Can trace recursive calls
- Understand call stack behavior

--------------------------------------------------

PHASE 4 — ARRAYS
Goal: Handle collections logically

- Topics:
  - Traversal
  - Searching
  - Rotation
  - Comparison

- Daily target: 4–5 questions
- Time: 10–12 days

Outcome:
- Strong array manipulation
- Confident with indexes

IMPORTANT:
After completing Phase 4 → START LEETCODE EASY

--------------------------------------------------

PHASE 5 — STRINGS
Goal: Strengthen string manipulation logic

- Topics:
  - Character handling
  - Word-based logic
  - Reversal and counting

- Daily target: 4 questions
- Time: 8–10 days

Outcome:
- Clear understanding of strings
- No fear of string problems

--------------------------------------------------

PHASE 6 — MIXED LOGICAL CHALLENGES
Goal: Combine all concepts

- Mix of:
  - Numbers
  - Strings
  - Arrays
  - Loops
  - Recursion

- Daily target: 3–4 questions
- Time: 10–12 days

Outcome:
- Interview-level confidence
- Real-world logical thinking

--------------------------------------------------
WHEN AND HOW TO START LEETCODE
--------------------------------------------------

Start LeetCode ONLY AFTER:
- Phase 4 is completed
- Arrays and loops are clear

Start With:
- Easy level only
- Topics:
  - Arrays
  - Strings

Avoid:
- Medium/Hard initially
- Copy-pasting solutions

--------------------------------------------------
DAILY STUDY ROUTINE (IDEAL)

30 minutes → Revise yesterday’s questions
90 minutes → Solve new questions
30 minutes → Debug and improve code

Total: ~2.5 hours/day
